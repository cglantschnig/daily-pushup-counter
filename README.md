# Daily Pushup Counter

Daily Pushup Counter is a small mobile-first workout app for running quick pushup sets in the browser. It generates a challenge, gives you an audio countdown, and stores completed sets in Convex under the signed-in Clerk user.

## What The App Does

- Starts a challenge flow from a simple home screen.
- Generates a random target between 5 and 10 reps.
- Starts the spoken `3, 2, 1, START` countdown directly from the challenge screen.
- Stores completed challenges in Convex for the signed-in Clerk user.
- Shows recent personal history for the current signed-in user.
- Lets the user switch between system, light, and dark themes.

## Routes

- `/` home screen with the primary challenge entry point
- `/challenge` generated workout summary, large `GO` button, and audio-assisted countdown
- `/history` recent challenge history from Convex
- `/settings` account and theme controls

## Local Development

```bash
pnpm install
pnpm convex:dev
pnpm dev
```

The dev server runs on `http://localhost:3000`.

Create `.env.local` before starting the app. It should include `CONVEX_DEPLOY_KEY` for deploys and the public `VITE_CONVEX_URL` generated by Convex for the browser client.

For Clerk auth, also provide:

- `VITE_CLERK_PUBLISHABLE_KEY`
- `CLERK_SECRET_KEY`
- `CLERK_JWT_ISSUER_DOMAIN`

You must also enable the Clerk-to-Convex integration so Clerk issues a `convex` JWT template that Convex can validate.

If Clerk returns `JWT template not found` or `No JWT template exists with name: convex`, the app is pointed at a Clerk instance that does not have a JWT template literally named `convex`. Create that template in Clerk and verify `VITE_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`, and `CLERK_JWT_ISSUER_DOMAIN` all belong to that same Clerk instance.

## Project Notes

- Challenge history is stored in Convex per signed-in Clerk user.
- User preferences are still stored in the browser.
- Spoken countdown depends on speech synthesis support in the current browser/device.
- The workout catalog is currently limited to pushups, but the route and workout helpers are already structured for expansion.

## Clerk + Convex rollout

1. Configure Clerk and Convex authentication outside the repo.
2. Delete the existing anonymous `challenges` rows from the current Convex deployment.
3. Run `pnpm convex:dev` to push the new Convex auth config and schema.
4. Start the app and verify sign-in, challenge save, and per-user history isolation.
